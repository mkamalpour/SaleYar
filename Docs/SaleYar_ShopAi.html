SaleYar — Business Intelligence for Iranian Shops
Offline · Private · Smart · 9‑stage pipeline · HTML dashboard · LLM Assistant · Never crashes
What Makes SaleYar Different
• Offline & private – no cloud, no fees
• Pure HTML dashboard – fast, responsive, no Gradio for owners
• LLM Assistant with Talk Mode – conversational or bullet‑point answers
• Markdown support – bold, lists, italics in AI responses
• Unified score – one number for ranking and purchase order
• MD5 cache per shop – same file = instant results
• Integer optimization – whole quantities via OR‑Tools
• Critical basket pairs – hard constraints in optimizer
How It Runs
Upload a file → MD5 hash → cache check.
Cache hit → results in <1 second.
Cache miss → runs 9 stages fresh, saves per‑shop models.
HTML dashboard polls API every 3 seconds. Never crashes.
Three Interfaces
🏪 HTML Owner Dashboard – 9 tabs, search, real‑time, LLM
🔧 Gradio Developer UI – full debugging, always bullet‑point LLM
⌨️ Terminal – command line, rich tables
📡 FastAPI – REST endpoints, API key auth
Advanced Capabilities
• Talk Mode – toggle between short conversational (max 10 words) and 4 bullet points
• Micro‑summary vs full summary – talk mode uses health, revenue, out‑of‑stock, top seller
• Tab guide for LLM – AI knows all dashboard tabs and can direct users
• Daily forecast arrays – 90 days of daily values for inventory simulation
• transaction_type support – sale / purchase / return_sale / return_purchase
• Auto‑detect seasonality – 7/30/365 days via numpy correlation

HTML Owner Dashboard – Full Specification
The main interface for shop owners. Replaces the old Gradio owner UI.

📑 9 Tabs

  • Overview – health score, revenue, profit, top 5 products, urgent out‑of‑stock, priority actions.
  • Purchase Orders – ALL products appear. Shows current stock, recommended qty (max 500), unit cost, total cost, plain reason. Copy list / export CSV.
  • Forecast – 30/60/90 day demand, confidence badges, segment icons, actionable recommendations.
  • Products – Ranked list (1,2,3…) with unified score, segment, revenue (M), profit (M), margin %, risk score (🟢🟡🔴), stock, last sale, risk explanation. Searchable.
  • Segments – Counts and product lists for Star, Reliable, Seasonal, Deadweight, Risky, Outlier. Action descriptions.
  • Basket Rules – Products bought together, confidence & lift. Critical pairs highlighted.
  • Inventory – Current stock, days of coverage, status (OUT/LOW/OK/OVERSTOCK), actions (ORDER NOW, ORDER THIS WEEK, RUN SALE).
  • Customers & ROI – Customer segments (Champions, Loyal, At‑Risk, Lost), shop ROI vs bank (23%) vs gold (35%), champion products.
  • Settings – API Base URL, API Key, AI Advisor Mode (Keyword / Real LLM), Talk Mode checkbox. Settings saved in localStorage.

🔍 Global Search & Real‑time Polling

Search bar filters products, inventory, orders, forecast. Upload → async job → polls every 3 seconds → auto‑update with loading overlay.

💬 LLM Assistant (Floating Robot)

  • Keyword mode – fast, offline, rule‑based (smartAnswer).
  • Real LLM mode – calls /llm/ask (LM Studio / Ollama).
  • Talk Mode toggle – ON → ultra‑short conversational answers (max 10 words), uses micro‑summary (health %, revenue/profit in millions, out‑of‑stock count, top seller). Greetings/thanks get no numbers. “How to find X?” → tells exact tab name. OFF → 4 bullet points, uses full summary.
  • Markdown support – bot messages rendered with marked.js (bold, italic, bullet lists).
  • Send button disabled during request, re‑enabled after response.
  • Fallback – if Real LLM fails, falls back to keyword advisor.
LLM Advisor – How It Works
Optional natural language answers using local LLM (LM Studio or Ollama).

🧠 Two Modes

  • Keyword mode – rule‑based, no internet, uses smartAnswer function. Fast and offline.
  • Real LLM mode – sends a compact summary to /llm/ask. Supports LM Studio (/api/v1/chat) and Ollama (/api/generate).

💬 Talk Mode

  • ON – AI answers in 1 very short sentence (max 10 words). Uses micro‑summary (health %, revenue/profit in millions, out‑of‑stock count, top seller). Greetings/thanks get no numbers. “How to find X?” → tells exact tab name. “What can you do?” → suggests 1‑2 actions.
  • OFF – AI gives 4 bullet points. Uses full summary (health, data quality, top 5 products, bottom 5, segments, inventory, forecast needs, basket rules, customer segments, priority actions, alerts, high‑risk products).

📋 Summary Construction

The micro‑summary is only 3‑4 lines: health %, revenue/profit in millions, out‑of‑stock count, top seller. The full summary is ~600‑800 tokens and includes everything. The LLM is also given a tab guide (list of all 9 tabs and what they contain) so it can direct users.
